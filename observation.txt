Explanation
The inventory value differs on every run because the OS process scheduler
decides when each of the 50 bots gets CPU time, and that order is unpredictable every time.
When two bots read inventory.txt at the same moment before either writes back, 
one bot result silently overwrites the other causing some purchases to be lost.
This is the classic TOC-TOU race condition.
